通用設置和事件頁（Event Pages）設置

ID (標題欄顯示)

每個事件都有自己的唯一ID，ID會在事件創建時根據這張地圖內創建順序被自動分配。事件ID被用於如指定事件使用特定變量的情況。

Name （名字）

地圖事件的名字，這個屬性僅被用在編輯器（不會影響遊戲）。ID將會被自動輸入作為初始名字。必要時可以改動名字以提高可讀性。

Note （備註）

給自己看的備註

Event Page ID（事件頁ID）

包含在這個事件當中的事件頁ID。通過點擊選項卡，你可以切換到你想要編輯的事件頁。

Event Page Control Buttons（事件頁控制按鈕）

窗口頂部的按鈕可以進行創建，刪除事件頁等操作。每個按鈕的功能羅列如下。

New Event Page（新建事件頁）
在當前正在編輯的事件頁後創建一個新的事件頁。每個額外事件頁創建時將會有一個數字自增ID。
Copy Event Page（複製事件頁）
複製當前事件頁的內容到剪貼板
Paste Event Page （粘貼事件頁）
從剪貼板複製的內容中添加（插入）一個事件頁到當前編輯的事件頁之後。事件頁創建時將會獲得一個自增ID。
Delete Event Page （刪除事件頁）
刪除當前編輯的事件頁。在這個事件頁之後的事件頁ID將會被減1.
Clear Event Page （清空事件頁）
清空當前正在編輯的事件頁內容（重置為初始狀態）

事件頁設置

Conditions（觸發條件）

這個事件頁的內容觸發的條件。觸發條件可以在[Switch]（開關）, [Variable]（變量）, [Self Switch]（獨立開關）, [Item]（物品） and [Actor]（角色）。設定特定的條件以滿足你的需求。

如果你希望事件無條件觸發，那麼不要設置任何條件。如果你設置多了多個條件，那麼事件僅會在所有條件都滿足的情況下事件才會被觸發。

如果有多個事件頁的條件被同時滿足了，那麼擁有最大ID的事件頁內容將會被觸發。反之，如果沒有事件頁的條件被滿足則不會有任何事件頁被觸發。

Switch（開關）
如果開關被設置為ON則此類型事件將會被觸發
Variable （變量）
指定變量大於給定值的時候事件將會被觸發。
Self Switch （獨立開關）
指定獨立開關變成ON狀態時被觸發
Item （物品）
一個隊伍擁有指定物品的時候事件將被觸發
Actor （角色）
指定角色在隊伍中時事件將被觸發

Image （圖像）

當事件出現在地圖上的時候圖像會被顯示。要指定圖像，雙擊事件框中心並在彈出的[Select an Image](選擇文件)窗口中點擊文件列表（左邊）和圖像（右邊）並選中。

如果你沒設置圖像，則事件圖像將不可見。在選擇圖像的窗口中選擇位於文件列表頂部的[(None)]可以刪除已經設置的圖像。

Autonomous Movement （自動移動）

用[Type]（類型）、[Speed]（速度）和[Frequency](頻率) 來指定地圖事件的移動方式。[速度]定義了移動的速度，數值越大移動越快。[頻率]定義了移動的週期，數值越大週期越短。[Type](類型)讓你指定下面四種移動方式之一。

Fixed(固定)
地圖事件將不會移動
Random （隨機）
地圖事件將會隨機四處移動
Approach （接近）
地圖事件將會自動接近玩家
Custom （自定義）
沿一條特定的路線移動。單擊[移動路線]的時候可以指定路線。更多信息請參見 [設置移動路線].

Options

可選的圖像的顯示方式。需要的話就選中特定的選項。

Walking（步行動畫）
移動的時候播放步行動畫。當事件是角色或動物等的時候使用。
Stepping（踏步動畫）
不動的時候播放踏步動畫。當事件是波浪或火焰等的時候使用。
Direction Fix （固定朝向）
禁止改變圖像的朝向，即使是在移動的時候。
Through（允許穿透）
允許它穿過一些無法通行的地形或事件。

優先級

在下面的選項中指定一個以確定事件和玩家的高低位置關係。如果事件和玩家可以重疊，那麼位置在上方的圖像優先顯示。

Below characters（人物下方）
玩家和其他角色可以從該事件上方通過。然而，如果在[Image]（圖像）中選擇了一個圖塊作為圖像，那麼玩家能否通行會根據這個圖塊的通行度來決定。
Same as characters（與人物同層）
將事件和玩家置於同層，禁止玩家走到該事件的位置。
Above characters（人物上方）
玩家和其他角色可以從該事件下方通過。

觸發條件

對於已經出現在地圖上了的事件，在下面的選項中指定一個以確定什麼時候執行事件的[Contents]（執行內容）。

Action Button(按確定鍵)
當玩家面對事件（如果允許重疊就是和玩家重疊）按下確定鍵的時候執行。
Player Touch(玩家接觸)
玩家碰觸到事件（如果允許重疊就是經過事件）的時候執行。這種情況下按確定鍵也會執行。
Event Touch（事件接觸）
當一個[Autonomous Movement](自主移動)的事件碰到了玩家（如果允許重疊就是經過玩家），以及[玩家接觸]的時候執行。
Autorun（自動執行）
當事件滿足出現條件，出現在地圖上後立刻執行。
並行處理
當事件滿足出現條件，出現在地圖上後立刻執行（週期性地循環處理執行內容）。

Contents （事件內容）

當滿足[Conditions]（觸發條件）滿足的時候執行的命令列表。更多信息請看接下來的項目："編輯事件指令"。

編輯內容

查看執行內容列表

在執行內容中，將遊戲中的處理以「[Event Commands](事件指令)」的方式進行編輯

事件指令會從頂部開始按順序處理，並且被應用到遊戲內容中。已被添加的事件指令會在行的最左端有一個[◆]的標記。以[:]開頭的行是它上方的事件指令的內容，或者是處理流程發生分支的位置

添加事件指令

要向列表中註冊一個事件指令，只需要雙擊有[◆]標記的行。在彈出的窗口中，選擇要使用的事件指令，然後定義想要的行為和設置即可（除了一小部分）。新雙擊的位置已經存在事件指令時，新添加的事件指令會插入到雙擊的位置。

上下文菜單的操作

已經被註冊了的事件指令有一個[◆]的標記。在這樣的行上面右擊會彈出一個菜單，裡面有一些常用操作如[Copy]（複製）和[Delete]（刪除）。這些項目的功能如下。

New（新建）
在選擇的指令的上一行插入一條新的事件指令
Edit（編輯）
編輯選中的事件指令。
Cut（剪切）
把當前的事件指令複製到剪貼板並刪除。
Copy（複製）
把當前的事件指令複製到剪貼板。
Paste（粘貼）
把剪貼板中的內容插入到選擇的位置。
Delete（刪除）
刪除所選的事件指令。
Select All（全選）
將列表中的所有事件指令全部選中。
Test（測試）
在遊戲中測試所有選中指令

選擇多行指令

單擊選中開始的那一行，然後按住[Shift]鍵單擊結束的那一行。這樣在這兩行之間的所有帶[◆]標記的指令都會被選中。在選中的事件指令（會變成藍色）上右鍵就可以彈出一個上下文菜單，你可以在裡面選擇一些操作，這些操作會影響所有被選中的指令。值得注意的是，如果選擇的指令在一個分支中，只有處於同一個分支的指令才能用這種方式選中。
