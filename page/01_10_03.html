Conditional Branch（條件分歧）

功能

指定各個條件分支處理的事件指令，因此一些特殊的過程只能在指定條件被滿足時運行。當[Create Else Branch（這以外的場合）]被選中時，當指定條件不滿足時，事件指令會落入另一個分支中。

設置

從下面的選項中選擇一個條件，同時你要指定執行的標準。

Switch（開關）
使用指定的開關值作為標準。你需要設置目標開關和它的狀態（ON / OFF）。
Variable（變量）
使用指定的變量值作為標準。你需要設置需要判斷條件的變量，和它比較的值，比較的方法。當需要使用其他變量作為比較的值時，選擇[Variable（變量）]，然後指定要引用的變量。
Self Switch（獨立開關）
使用指定的獨立開關的狀態作為標準。你需要設置目標獨立開關和它的狀態（ON / OFF）。
Timer（計時器）
使用計時器的剩餘時間作為標準。你需要指定剩餘的時間和標準（多於/少於）。
Actor（角色）
使用指定的角色作為標準。選擇目標角色和要參考的內容（in party/name/class/skill/weapon/armor/state[在隊伍中/名稱/職業/技能/武器/防具/狀態]），有些內容還有額外的標準。
Enemy（敵人）
在戰鬥中，使用指定敵人作為標準。選擇目標敵人和執行的標準（出現/狀態）。這僅在戰鬥事件中有效。
Character（地圖上的角色）
使用地圖上的主角或地圖事件的方向作為標準。你需要指定目標角色和方向。這僅在地圖事件中有效。
Vehicle（交通工具）
使用主角正乘坐的交通工具（小型船/大型船/飛空艇）作為標準。
Gold（金錢）
使用隊伍擁有的金錢作為標準。你需要指定數量和比較的方法（多於/少於）。
Item（物品）
使用角色是否持有某特定物品作為標準。
Weapon（武器）
使用隊伍是否擁有某特定武器作為標準。當選項[Include Equipment（包含裝備）]選中時，主角身穿的裝備也會被考慮在內。
Armor（防具）
使用隊伍是否擁有某特定防具作為標準。當選項[Include Equipment（包含裝備）]選中時，主角身穿的裝備也會被考慮在內。
Button（按鈕）
使用特定按鈕是否被按下作為標準。
Script（腳本）
高級用戶使用：使用腳本的執行結果作為標準。
右鍵點擊可以顯示菜單，選擇[Plugin Help...（插件幫助）]來顯示插件幫助。
（2016.03.01 更新）

Loop（循環）

功能

在[Loop（循環）]和[Repeat Above（以上往復）]之間添加事件指令，這些指令會被反覆執行。甚至你可以運行「多重循環」，也就是循環中間嵌套另一層循環。

注記

  • 在[Loop（循環）]和[Repeat Above（以上往復）]之間的事件指令會無限地重複執行。你可能需要其他事件指令（如[Break Loop（中斷循環）]）來中斷重複的過程。

Break Loop（中斷循環）

功能

中斷[Loop（循環）]指令指定的循環過程。此指令執行後，事件將會跳到[Repeat Above（以上往復）]命令後面的那一行繼續執行。此處不需要設置參數。

Exit Event Processing（中斷事件處理）/h2>

功能

中斷事件內容的處理。這命令後面的的指令將不會被執行。此處不需要設置參數。

Common Event（公共事件）

功能

觸發公共事件。公共事件執行完畢後，遊戲會繼續執行本事件餘下的內容。

設置

Common Event（公共事件）
指定要執行的公共事件。

Label（標籤）

功能

為指定過程創建一個跳轉的位置。可以通過使用[Jump to Label（標籤跳轉）]指令跳到這個位置處執行。

設置

Label Name（標籤名稱）
指定標籤的名稱。

Jump to Label（標籤跳轉）

功能

事件指令執行到此行時，會自動跳轉到指定的標籤位置繼續向下執行。

設置

Label Name（標籤名稱）
指定要跳轉到的標籤名。你只能指定本事件頁中的標籤。

Comment（註釋）

功能

在事件內容中添加註釋。這不會影響遊戲。

設置

Comment（註釋）
註釋的內容文字。
右鍵點擊可以顯示菜單，選擇[Plugin Help...（插件幫助）]來顯示插件幫助。
（2016.03.01 更新）
